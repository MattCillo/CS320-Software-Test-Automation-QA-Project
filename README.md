# CS320-Software-Test-Automation-QA-Project
Ensuring that my code is both functional and secure comes down to disciplined testing. In the past, I would often only make sure the program worked when everything was entered perfectly. Working on projects like the contact and task services taught me that real functionality is proven when things go wrong. I now make it a point to write tests that deliberately try to break my own rules, such as entering missing values or data that exceeds the allowed limits. Catching these issues early helps me prevent simple bugs from becoming serious security vulnerabilities or system crashes. It is a humbling process because it constantly reminds me that my initial code is rarely perfect, but testing provides a safety net to help me improve.

Interpreting user needs and incorporating them into a program starts with active listening rather than technical problem-solving. A system can have flawless code, but if it does not fit naturally into the end user's daily workflow, it is a failure. When reviewing case studies and designing interfaces, I learned the importance of asking the right questions to understand the core business problem. I try to put myself in the user's shoes to figure out what they actually need to accomplish. From there, I translate those everyday human needs into clear, step-by-step requirements for the software.

My approach to designing software has become much more patient and methodical. Instead of jumping straight into my code editor, I start by stepping back and planning the architecture visually. I use models and diagrams to map out how different objects and processes will interact with one another before any code is written. Once I have a clear blueprint, I approach the actual building phase by breaking the system down into small, isolated pieces. This keeps the project from feeling overwhelming and makes finding errors much easier. I approach design with the mindset that I am building a clean, organized structure that other developers and I can easily understand and maintain.
